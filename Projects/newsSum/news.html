PyCA cryptography 2.5 released

b'PyCA cryptography 2.5 has been released to PyPI. cryptography includes both\nhigh level recipes and low level interfaces to common cryptographic\nalgorithms such as symmetric ciphers, asymmetric algorithms, message\ndigests, X509, key derivation functions, and much more. We support Python\n2.7, Python 3.4+, and PyPy.\n\nChangelog (https://cryptography.io/en/latest/changelog/#v2-5):\n\n* BACKWARDS INCOMPATIBLE: U-label strings were deprecated in version 2.1,\nbut this version removes the default idna dependency as well. If you still\nneed this deprecated path please install cryptography with the idna extra:\npip install cryptography[idna].\n* BACKWARDS INCOMPATIBLE: The minimum supported PyPy version is now 5.4.\n* Numerous classes and functions have been updated to allow bytes-like\ntypes for keying material and passwords, including symmetric algorithms,\nAEAD ciphers, KDFs, loading asymmetric keys, and one time password classes.\n* Updated Windows, macOS, and manylinux1 wheels to be compiled with OpenSSL\n1.1.1a.\n* Added support for SHA512_224 and SHA512_256 when using OpenSSL 1.1.1.\n* Added support for SHA3_224, SHA3_256, SHA3_384, and SHA3_512 when using\nOpenSSL 1.1.1.\n* Added support for X448 key exchange when using OpenSSL 1.1.1.\n* Added support for SHAKE128 and SHAKE256 when using OpenSSL 1.1.1.\n* Added initial support for parsing PKCS12 files with\nload_key_and_certificates().\n* Added support for IssuingDistributionPoint.\n* Added rfc4514_string() method to x509.Name,\nx509.RelativeDistinguishedName, and x509.NameAttribute to format the name\nor component an RFC 4514 Distinguished Name string.\n* Added from_encoded_point(), which immediately checks if the point is on\nthe curve and supports compressed points. Deprecated the previous method\nfrom_encoded_point().\n* Added signature_hash_algorithm to OCSPResponse.\n* Updated X25519 key exchange support to allow additional serialization\nmethods. Calling public_bytes() with no arguments has been deprecated.\n* Added support for encoding compressed and uncompressed points via\npublic_bytes(). Deprecated the previous method encode_point().\n\n-Paul Kehrer (reaperhulk)\n\n'

ANN: gcc-python-plugin 0.17

b'gcc-python-plugin is a plugin for GCC 4.6 onwards which embeds the\nCPython interpreter within GCC, allowing you to write new compiler\nwarnings in Python, generate code visualizations, etc.\n\nThis release adds support for gcc 9 (along with continued support for\ngcc 4.6, 4.7, 4.8, 4.9, 5, 6, 7, and 8).\n\nUnfortunately, the reference-count checker no longer works for gcc 7\nonwards, so it is disabled when embedded in those builds of gcc.\n\nAdditionally, this release contains the following improvements:\n\n* the plugin can now be built in a separate build directory from the\n  source directory (thanks to Tom de Vries)\n\n* gcc-with-cpychecker gained a --cpychecker-verbose option\n\nTarball releases are available at:\n  https://github.com/davidmalcolm/gcc-python-plugin/releases\n\nPrebuilt-documentation can be seen at:\n  http://gcc-python-plugin.readthedocs.org/en/latest/index.html\n\nThe plugin is Free Software, licensed under the GPLv3 or later.\n\nEnjoy!\nDave Malcolm\n\n\n'
